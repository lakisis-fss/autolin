# 02_ 보안 모듈 정밀 로드 상태 진단 보고서

## 개요
안티 치트 시스템에 의해 은닉된 보안 모듈 `Formula.Security.dll`의 실제 로드 상태와 이를 점유 중인 프로세스를 정밀 진단한 결과입니다.

---

## 1. 정밀 진단 파일 정보
*   **파일명**: `Formula.Security.dll`
*   **파일 경로**: `C:\Program Files (x86)\NCSOFT\Purple\2.26.330.16\Formula.Security.dll`
*   **전자 서명**: `NCSOFT Corporation` (유효함)
*   **특징**: 퍼플 플랫폼의 핵심 보안 라이브러리로, 게임 실행 무결성 및 인메모리 감시 기능을 수행함.

## 2. 프로세스 및 핸들 점유 진단 결과
PowerShell을 통한 다각도 진단 결과는 다음과 같습니다.

### 2.1 프로세스 모듈 은닉 상태 (Module Hiding)
- **진단 대상**: `LC.exe` (PID 4528), `purple-agent.exe` (PID 18624)
- **결과**: `Get-Process` 및 `tasklist /m` 쿼리 시 해당 모듈 정보가 **`N/A`** 또는 빈 값으로 반환됨.
- **분석**: 이는 커널 드라이버(`KslD.sys`)가 유저 모드 모듈 열거 API를 후킹(Hooking)하여 해당 파일의 로드 정보를 의도적으로 보이지 않게 처리하고 있는 전형적인 안티 디버깅 기법입니다.

### 2.2 파일 핸들 탐색 (Handle Search)
- **진단 방식**: 전체 프로세스를 대상으로 `Formula.Security.dll` 파일을 열고 있는 핸들(Handle) 추적 실험.
- **결과**: 직접적인 핸들 정보 탐색이 차단됨. 하지만 게임 실행(`LC.exe`) 및 보안 에이전트(`purple-agent.exe`)가 활성화된 시점에 해당 파일이 시스템적으로 보호(In-use) 상태에 있음을 확인했습니다.
- **결론**: 해당 모듈은 **메모리에 로드된 상태로 활발하게 동작 중**이나, 외부 분석 도구로부터 자신을 숨기기 위해 드라이버 레벨에서 은닉 처리가 되어 있습니다.

## 3. 개발 우회 전략 및 시사점
상세 진단 결과를 바탕으로 다음과 같은 개발 전략을 도출했습니다.

- **모듈 직접 접근 불가**: `Formula.Security.dll`을 직접 패치하거나 메모리에서 언로드하는 시도는 안티 치트 드라이버에 의해 감지되어 즉시 계정 제재로 이어질 가능성이 높습니다.
- **외부 자동화 권장**: 게임 내부 모듈을 조작하기보다는, 시스템 외부(화면 인식 및 하드웨어 입력 시뮬레이션)에서 동작하는 방식을 택하는 것이 가장 안전한 우회 경로임을 다시 한번 확증했습니다.
