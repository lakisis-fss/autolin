# 리니지 클래식 및 퍼플 안티 치트 감시 체계 진단 보고서

## 개요
본 보고서는 2026년 4월 5일, 로컬 PC에서 실행 중인 리니지 클래식(Lineage Classic) 및 퍼플(Purple) 플랫폼의 안티 치트 시스템 동작 방식과 모니터링 메커니즘을 분석한 결과입니다.

---

## 1. 커널 계층 (Kernel Layer) 분석
보안 드라이버가 운영체제 최하단에서 시스템 이벤트를 감시하고 있습니다.

- **식별된 보안 드라이버**: `KslD.sys`
    - **동작**: 커널 모드에서 로드되어 실행 중.
    - **주요 역할**: 프로세스 메모리 보호, 비정상적인 디버깅 시도 차단, 하드웨어 입력 모니터링.
- **파일 시스템 미니필터**: 
    - `0x80070005 (Access Denied)` 오류가 확인됨. 이는 보안 드라이버가 파일 시스템 접근 권한을 제어하고 외부 프로세스의 실시간 감시를 방해하고 있음을 나타냄.

## 2. 유저 모드 (User Mode) 프로세스 감시
플랫폼 수준에서 다수의 감시 에이전트가 동작하고 있습니다.

- **보안 에이전트**: `purple-agent.exe` (PID: 18624)
    - 게임 실행 중 상주하며 비정상적인 외부 툴 및 입력 패턴을 실시간 감시.
- **자가 보호 기술 (Self-Protection)**:
    - `tasklist /m` 쿼리 시 `LC.exe` 및 `purple-agent.exe`의 로드된 모듈 정보를 `N/A`로 은닉.
    - 외부 분석 도구가 게임의 내부 라이브러리(DLL) 구조를 파악하는 것을 차단.
- **원격 및 캡처 감시**:
    - `purpleonx.exe`, `purpleonp.exe`: 퍼플 온(Purple On) 기능을 이용한 화면 스트리밍 및 원격 제어 탐지.

## 3. 통신 및 무결성 메커니즘
- **IPC 하트비트 (Inter-Process Communication)**:
    - `netstat` 분석 결과, `purple-agent`와 게임 프로세스 간에 로컬 루프백(`[::1]:63911`)을 통한 실시간 데이터 교환 확인.
    - 보안 상태 및 무결성 정보를 주기적으로 검증하는 하트비트(Heartbeat) 신호로 추정됨.

## 4. 분석 결론 및 대응 전략

### 4.1 화면 인식 기반 자동화 (Screen-based Automation)
- **위험 요소**: `Purple On` 관련 모듈이 표준 화면 캡처 방식을 감시함.
- **대응**: Windows Graphics Capture API 또는 하드웨어 기반 캡처 보드 사용 권장.

### 4.2 입력 시뮬레이션 (Input Simulation)
- **위험 요소**: 소프트웨어 API(`SendInput` 등)는 `KslD.sys` 드라이버에 의해 패턴 분석 및 차단될 가능성이 높음.
- **대응**: Arduino HID 에뮬레이션 또는 커널 드라이버 수준의 입력 장치 시뮬레이션 권장.

### 4.3 패턴 우회 (Pattern Evasion)
- **동적 가변성**: 마우스 이동 궤적에 베지어 곡선(Bezier Curve)을 적용하고 클릭 간격과 위치에 무작위성(Randomization)을 부여하여 패턴 분석 우회 필요.
