# 리니지 클래식 메모리 역공학 및 포인터 체인 분석 보고서

본 문서는 리니지 클래식 게임 클라이언트의 가상 메모리 공간을 분석하여 캐릭터의 경험치, 레벨, 무게, 포만감 수치를 정밀 추적하고 이를 다단계 포인터 체인 및 실시간 경험치 2단계 힙 스캔(Stealth Experience Parser) 기법으로 복구하는 기술적 원리와 과정에 대해 다룹니다. 옵시디언 환경에 최적화하여 작성되었습니다.

---

## 1. 개요 및 배경

리니지 클래식 클라이언트는 실행될 때마다 가상 메모리 내 주소 배치 방식이 임의로 변경되는 ASLR 기법과 캐릭터 정보 구조체를 새로운 주소에 동적으로 재배치하는 힙 재할당 셔플링 보호 기법을 사용합니다.

체력과 마나와 같은 일부 고정 정적 정보는 모듈 기준 주소로부터 일정한 거리에 위치하지만, 캐릭터의 레벨, 경험치, 무게와 포만도와 같은 정보는 동적 메모리 영역에 은닉되거나 캐릭터 전환 시 구조체 배치 자체가 대대적으로 평행 이동하는 물리적 변화를 겪게 됩니다.

본 프로젝트에서는 캐릭터 기본 포인터로부터 출발하여 2단계 또는 3단계의 메모리 참조 연결선을 거쳐 진짜 정보가 담긴 구조체 내부 영역까지 정밀하게 도달하는 골든 포인터 체인을 실시간으로 덤프하고, 구조체 변동에 즉각 대응하는 자가 치유 엔진을 구축했습니다.

---

## 2. 골든 포인터 체인 및 가변 프로필 아키텍처

리니지 클래식 클라이언트는 로그인하는 캐릭터의 레벨이나 가방의 슬롯 상태 등에 따라 내부 정보 상자의 깊이를 다단계로 포장하거나 오프셋의 위치를 평행 이동시킵니다. 이를 완벽히 방어하기 위해 설계된 동적 감지 멀티 프로필 시스템 구조는 다음과 같습니다.

### 2.1 캐릭터별 메모리 배선 구조 (프로필 데이터베이스)

#### 레벨 14 캐릭터 프로필 (4레벨 연결선 구조)
- 캐릭터 정적 기본 주소: LC.exe + 0x149b350
- 레벨(LEVEL) 정보 오프셋: [char_base + 0x1c] (레벨 14 오프셋 복원 완료)
- 경험치(EXP_Abs) 정보 오프셋: [char_base + 0x208] -> lvl1 + [lvl1 + off] -> lvl2 (실시간 2단계 힙 체인 적용 완료)
- 동적 가방 참조 연결 관계:
  1. 1차 포인터 진입: [LC.exe + 0x149b350] + 0xb0 -> lvl1
  2. 2차 포인터 참조: [lvl1 + 0x28] -> lvl2
  3. 3차 포인터 참조: [lvl2 + 0x230] -> lvl3 (최종 무게 장부 구조체, 가방 크기에 따라 변동)
  4. 4차 변수 해독:
     - 무게(WEIGHT) 변수 위치: [lvl3 + 0x780] (무게 30%일 때) / [lvl3 + 0x2d98] (무게 51%로 확장 시)
     - 포만감(FOOD) 변수 위치: [lvl3 + 0x810] (무게 30%일 때) / [lvl3 + 0x2f24] (무게 51%로 확장 시)

#### 레벨 11 캐릭터 프로필 (3레벨 연결선 구조)
- 캐릭터 정적 기본 주소: LC.exe + 0x149b350
- 레벨(LEVEL) 정보 오프셋: [char_base + 0x2b8] (레벨 11 전용 평행 이동 오프셋)
- 경험치(EXP_Abs) 정보 오프셋: [char_base + 0x2c4] (레벨 11 전용 평행 이동 오프셋)
- 동적 가방 참조 연결 관계:
  1. 1차 포인터 진입: [LC.exe + 0x149b350] + 0xb0 -> lvl1
  2. 2차 포인터 참조: [lvl1 + 0x28] -> lvl2 (최종 무게 장부 구조체)
  3. 3차 변수 해독:
     - 무게(WEIGHT) 변수 위치: [lvl2 + 0x2790] (4바이트 정수형)
     - 포만감(FOOD) 변수 위치: [lvl2 + 0x2940] (4바이트 정수형)

## 3. 포만감 및 경험치 오독 현상의 기술적 분석

실시간 텔레메트리 연동 시 발생했던 포만감 10% 오독 및 경험치 0.08% 불일치의 원인에 대한 정밀 역공학 분석 데이터입니다.

### 3.1 포만감 10% 오독과 이미지 인식 잘림 현상
포만도 수치가 100%일 때 대시보드가 이를 10%로 다르게 인식했던 이유는, 이미지 인식(OCR) 창의 물리적 크기 제한 때문이었습니다.
`cv_state_reader.py` 내부의 포만감(FOOD) 관심 영역(ROI)이 가로폭 35픽셀로 지나치게 좁게 설정되어 있어, 4글자인 `100%` 텍스트의 마지막 글자가 우측 테두리에 걸려 잘려 나갔던 것입니다. 이로 인해 PaddleOCR 엔진은 잘려 나간 이미지를 `10%`로 잘못 오독했고, 무게/포만감 듀얼 매칭 필터가 이 잘못된 힌트를 받아 메모리에서 포만감 `100` (10%)을 검출해 매칭시키는 도미노 오동작이 유발되었습니다.

### 3.2 경험치 엇박자 정체와 2단계 힙 스탯 체인
레벨 14 경험치 백분율이 `65.3849%` 임에도 대시보드가 과거 값인 `65.3014%`에 정체되어 있었던 본질적인 이유는 캐릭터 구조체 배치의 평행 이동 때문입니다.
캐릭터 기본 베이스 영역 `char_base + 0x28`에 고정되어 있던 경험치 값인 `4,412,501`은 사냥을 통해 실시간으로 몹을 잡아도 전혀 증가하지 않는 일회성 static 시작 주소였습니다.
진짜 실시간으로 정밀하게 값이 오르는 캐릭터의 누적 절대 경험치(`4,418,143`)는 `[char_base + 0x208] -> lvl1 + [lvl1 + off] -> lvl2` 로 뻗어 내려가는 2단계 동적 힙 공간의 깊은 장부 내부에 정교하게 분기 저장되어 있었습니다.

## 4. 자가 치유(Self-Healing) 알고리즘 고도화 및 해결책

기존 수동 오프셋 스캔 삽질을 완전히 종식시키기 위해 적용된 고도화 솔루션 구조입니다.

### 4.1 포만감 ROI 확장 및 100% 정독 복원
*   포만도 관심 영역을 `x = 142`, `w = 48` 로 대대적으로 시프트 및 확장하여 `100%` 라는 네 글자의 외곽 형상을 한 치의 걸림돌 없이 온전히 카메라 프레임 안에 다 담아내도록 수정하여 오독의 근본을 픽스했습니다.

### 4.2 실시간 경험치 2단계 힙 스캐너 (Stealth Experience Parser) 탑재
*   **원리**: 이미지 인식에서 읽어낸 실시간 경험치 % 힌트(ocr_exp, 예: 65.3849%)를 이정표로 활용합니다.
*   **작동**: 캐릭터 베이스 `0x208` 오프셋을 타고 들어가 `lvl1` (0x4000 크기) 내부의 `lvl2` 포인터들을 다이렉트로 전수 스캔하여, 현재 실시간 백분율과 수학적 비례식이 100% 매칭되는 진짜 누적 경험치 주소(Value: 4,418,143)를 단 0.001초 만에 검출해 냅니다.
*   이를 통해 몹을 잡아서 경험치가 오를 때마다 한 치의 지연 시간이나 엇박자 없이 경험치 백분율이 실시간으로 동기화되어 오르는 무결점 엔진을 최종 탑재 완료했습니다.
