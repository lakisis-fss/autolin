# 03_ 리니지 클래식 화면 인식 테스트 결과 보고서

## 개요
자동화 프로그램 개발의 첫 단계인 '화면 인식(Screen Recognition)' 가능 여부를 검증하기 위해, 현재 실행 중인 리니지 클래식 창을 프로그램적으로 캡처하고 데이터의 유효성을 분석한 결과입니다.

---

## 1. 테스트 환경 및 방법
*   **환경**: 리니지 클래식 창 모드 (Always on Top 설정)
*   **사용 언어/라이브러리**: Python 3, `mss`, `pywin32`, `numpy`, `opencv-python`
*   **테스트 스크립트**: `test_capture.py`
    *   `win32gui`를 사용하여 "Lineage Classic" 창의 HWND 및 좌표를 동적으로 탐색.
    *   `mss`를 이용해 해당 영역의 메모리 데이터를 캡처 후 PNG로 저장.

## 2. 테스트 결과
*   **윈도우 탐지**: 성공 (Window Title: `Lineage Classic - 26.03.31.2001 [LIVE] - Login ...`)
*   **캡처 좌표**: `L=43, T=50, R=1339, B=1049` (Size: `1296x999`)
*   **데이터 유효성**: **정상 (데이터 포함됨)**
    *   캡처된 파일 크기: **1,597,515 Bytes (~1.5MB)**
    *   안티 치트(KslD.sys 등)에 의한 **검은색 화면(Black Screen) 차단 증상이 발견되지 않음**.
*   **저장 파일**: [03_test_capture.png](file:///c:/Users/drspi/Downloads/Project/Autolineage/docs/03_test_capture.png)

## 3. 결론 및 향후 계획
### 3.1 기술적 결론
리니지 클래식은 창 모드 실행 시 표준 GDI/DirectX 캡처 인터페이스를 차단하지 않는 것으로 확인되었습니다. 이로써 **OpenCV를 활용한 이미지 매칭(몬스터 탐지, 상태창 인식)이 기술적으로 가능함**을 확증했습니다.

### 3.2 향후 구현 방향
1.  **이미지 매칭 엔진**: 몬스터, HP 바, 스킬 아이콘 등의 샘플 이미지를 사용하여 특정 객체를 화면에서 찾아내는 로직 구현.
2.  **자동 사냥 프로토타입**: 인식된 객체의 위치를 기반으로 마우스/키보드 입력을 시뮬레이션하는 단계로 진입.
