# 무게 소수점(Float) 메모리 추적 스크립트 제작 안내

## 1. 개요
포만도 수치가 정수가 아닌 소수점(Float)으로 저장되어 있다는 점을 응용하여, 무게 수치 역시 동일한 원리로 찾아낼 수 있도록 `find_weight_float.py` 스크립트를 새롭게 작성했습니다. 무게 또한 화면에는 35%처럼 정수로 보이지만 메모리 상에서는 35.1234% 처럼 소수점으로 관리될 확률이 매우 높습니다.

## 2. 작동 원리
포만도를 찾았던 방식과 완벽하게 동일한 논리를 따릅니다.
1. **1차 스냅샷**: 메모리를 전체 스캔하여 0.0부터 200.0 사이의 소수점 값을 모두 복사합니다. (무게는 100%를 초과할 수 있어 범위를 조금 넓게 200.0으로 설정했습니다)
2. **변화 필터링**: 게임 내에서 아이템을 줍거나 버려서 무게를 변화시킨 후 스크립트에 그 변화(증가/유지/감소)를 입력합니다.
3. **AOB 추출**: 조건을 만족하지 않는 메모리 주소들을 계속 걸러내면 마지막에 진짜 무게 주소 1개만 남습니다. 여기서 16바이트의 AOB 지문을 추출하여 자동 오프셋 스캐너에 적용할 수 있습니다.

## 3. 스크립트 사용 방법
1. 게임에 접속한 상태에서 `/scratch` 폴더 내에 새로 제작된 `find_weight_float.py`를 실행합니다.
2. 스크립트가 1차 메모리 스냅샷을 찍을 때까지 대기합니다.
3. 스냅샷이 완료되면, 게임 화면에서 물약을 하나 줍거나 버려서 무게 퍼센트를 바꿉니다.
4. 스크립트 화면에 방금 한 행동에 맞춰 1번(증가), 2번(유지), 3번(감소)을 선택합니다.
5. 이 과정을 주소가 1개에서 2개 정도 남을 때까지 반복하면 성공적으로 AOB 지문을 얻을 수 있습니다.

이 스크립트를 통해 얻어낸 AOB 패턴을 기존의 엔진에 적용하면, 게임 패치로 인해 주소가 바뀌더라도 무게 수치를 항상 고정적으로 읽어올 수 있게 됩니다.
